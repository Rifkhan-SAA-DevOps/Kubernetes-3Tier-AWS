7. Rolling Updates and Rollbacks
    Now you will learn how real companies deploy new versions without stopping the app.
    Kubernetes Deployments support rolling updates, where old Pods are gradually replaced by new Pods so the application can remain available during the update.

    minikube image load backend-three-tier-app:v2
    minikube image ls | findstr three-tier-app
    kubectl set image deployment/backend backend=backend-three-tier-app:v2 -n three-tier-app
    kubectl rollout status deployment/backend -n three-tier-app
    kubectl rollout history deployment/backend -n three-tier-app

    kubectl set image deployment/backend backend=k8s-backend-demo:broken -n three-tier-app


    kubectl rollout status deployment/backend -n three-tier-app
    kubectl rollout undo deployment/backend -n three-tier-app


8. Kubernetes Health Checks: readinessProbe, livenessProbe, startupProbe

    Probe	        Meaning	Kubernetes                  action
    readinessProbe	Is this app ready for traffic?	    Remove Pod from Service traffic if failed
    livenessProbe	Is this app still alive?	        Restart container if failed
    startupProbe	Has this app started successfully?	Wait before running liveness/readiness checks

13. Helm from scratch using your full-stack project
    Helm is used to package Kubernetes YAML files into reusable applications called Charts. 
    Instead of applying many YAML files one by one, Helm lets you install, upgrade, rollback, and uninstall an application as one release. 
    Helm’s official docs describe Charts as packages that help define, install, and upgrade Kubernetes applications.

    -------Why do we need Helm?----------

        Until now, your app has many files:

            namespace.yaml
            backend-deployment.yaml
            backend-service.yaml
            frontend-deployment.yaml
            frontend-service.yaml
            frontend-ingress.yaml
            backend-pvc.yaml
            postgres-secret.yaml
            postgres-service.yaml
            postgres-statefulset.yaml
            backend-db-config.yaml
            backend-db-secret.yaml



    helm version

    helm create fullstack-chart

    helm template fullstack-app ./fullstack-chart => Before installing, check whether Helm can render valid YAML.

    helm lint ./fullstack-chart

    helm install fullstack-app ./fullstack-chart

    helm list -A

    helm status fullstack-app

    helm upgrade fullstack-app ./fullstack-chart

    helm upgrade fullstack-app ./fullstack-chart --set frontend.replicas=3

    helm upgrade fullstack-app ./fullstack-chart -f values-dev.yaml

    helm history fullstack-app

    helm rollback fullstack-app 1

    helm uninstall fullstack-app
